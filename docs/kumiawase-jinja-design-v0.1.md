# 組み合わせ神社 設計書 Ver.0.1

作成日: 2026-06-27

## 位置づけ

組み合わせ神社は、水曜会専用ではなく、囲碁・将棋・ボードゲーム・カードゲーム・地域交流会などで使える汎用の組み合わせツールとして設計する。

水曜会向けの使い方は、汎用機能の上に乗るプリセットのひとつとして扱う。

## 基本思想

アプリ全体の流れは「集める → 育つ → 思い出になる」。

- 花図鑑: 参加する → 花が咲く → 妖精に会う
- 書庫: 称号を得る → 本が増える → 冒険日誌に残る
- 組み合わせ神社: 参加者を集める → 願いを選ぶ → お告げが出る → 記録に残る

組み合わせ神社は、外の世界の人も使える入口になる。花図鑑・冒険者カード・書庫は水曜会の物語を育てる場所、組み合わせ神社は誰でも使える便利な道具として役割を分ける。

## 中核モデル

組み合わせ神社は、次の要素で構成する。

- イベント: 何の会か。大会、交流会、卓分けなど。
- 参加者: 名前、レベル、所属、参加可否など。
- 役割: プロ、先生、運営、初参加、子ども、保護者、ゲームマスター、初心者など。
- 形式: 個人戦、ペア戦、団体戦、スイス式、総当たり、ランダム交流、卓分けなど。
- 条件: 組み合わせ時に守りたい願いごと。
- 結果: 実際の対戦、卓、席、チーム。
- 説明: 神様のひとこと、注意表示、条件の達成度。
- 記録: 過去の組み合わせを見返せる場所。

## 役割

「先生だから特別」ではなく「役割だから特別」と考える。

例:

- 水曜会: 先生、生徒、プロ、初参加、運営
- ボードゲーム会: 初心者、経験者、ゲームマスター、インスト係
- カードゲーム会: 初参加、上級者、ジャッジ、招待選手

役割を汎用化することで、競技やイベントの種類が増えても同じ仕組みで扱える。

## 条件カード

条件は固定の大きな処理ではなく、追加できる「条件カード」として扱う。

例:

- 同じ相手を避ける
- レベル差を小さくする
- 役割を分散する
- 同じ所属を避ける
- 勝ち同士・負け同士を当てる
- 初参加を孤立させない
- 希望ペアを優先する
- 親子・夫婦を離す
- 前回休憩した人を優先する
- 女性を均等配置する
- プロ同士を避ける

各条件カードには強さを持たせる。

- 必須: 絶対に破らない
- なるべく: 可能な範囲で守る
- 注意だけ: 結果に警告として表示する

この3段階にすると、条件が多すぎて組み合わせ不能になる問題を避けやすい。

## 神様のひとこと

組み合わせ結果のあとに、なぜその組み合わせになったかをやわらかく伝える。

例:

- 今日は初参加の方がいるので、交流しやすさを優先しました。
- 同じ相手を避けることを一番大事にしました。
- すべての条件を満たせなかったため、レベル差を少し広げています。
- 勝ち同士を優先しつつ、同じ所属が続かないようにしました。

専門的な条件一覧だけでなく、主催者が納得しやすい説明として使う。

## 注意表示

結果画面には、必要に応じて注意を出す。

例:

- この2人は前回も対戦しています。
- レベル差が大きめです。
- この卓だけ同じ所属が集まっています。
- このチームには初心者が複数います。

注意表示は失敗ではなく、主催者が手直しするための判断材料とする。

## 画面構成案

1. イベントを作る
   - 会の名前、競技、人数、回数、用途を決める。

2. 参加者を入れる
   - 名前、レベル、所属、参加可否を管理する。

3. 役割をつける
   - プロ、初心者、運営、先生、ゲームマスターなどを設定する。

4. 形式を選ぶ
   - 個人戦、ペア戦、団体戦、スイス式、卓分けなどを選ぶ。

5. 願いごとを選ぶ
   - 条件カードを選び、必須・なるべく・注意だけを決める。

6. お告げを見る
   - 組み合わせ結果、神様のひとこと、注意表示を見る。

7. 手直しする
   - 入れ替え、固定、再生成、条件をゆるめる。

## プリセット

プリセットは、汎用機能に初期設定を入れる仕組みとして扱う。

例:

- 水曜会 指導碁
- 囲碁 個人戦
- 囲碁 ペア碁
- 将棋 交流会
- ボードゲーム 卓分け
- カードゲーム スイス式

水曜会の場合:

- 先生 = 役割
- プロ = 役割
- 指導碁 = 形式
- 先生の輪 = 水曜会プリセット固有の達成記録

## 記録

過去の組み合わせは見返せるようにする。

名称候補:

- 御神託帳
- 祭事記録
- お告げの記録

記録には、イベント名、日時、形式、参加者、条件、結果、神様のひとこと、注意表示を残す。

例:

- 去年のクリスマス大会
- 忘年会団体戦
- 月例ボードゲーム会

## 実装優先度

今は実装しない。まずは設計を保存する。

現在のアプリ作業では、花図鑑・冒険者カード・書庫の仕上げを優先する。組み合わせ神社は次の大きな機能候補として、設計を育てる。
