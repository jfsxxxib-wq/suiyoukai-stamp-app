# 完成品保管庫ルール

このルールは、完成品保管庫へ花スタンプ・妖精スタンプ・確認用一覧を入れる時に毎回守ります。

## 基本方針

- 完成品保管庫へ入れる前に、必ず画面で確認します。
- 「成功品」と確認できたものだけを完成品保管庫へ入れます。
- 迷いがあるもの、作りかけ、候補、失敗品は完成品保管庫へ入れません。
- 完成品保管庫に入れる時は、あとで見ても分かる名前を付けます。

## スタンプ本体

- アプリで使うスタンプ本体には、画像の中に名前を入れません。
- 花名・妖精名・説明文などの文字は入れません。
- できるだけ背景なし、または透明背景PNGにします。
- 透明化すると細い線や羽が欠けそうな時は、先に薄い背景版で確認します。
- ファイル名には名前を入れます。

例:

- `hagi-cat-fairy-transparent.png`
- `lotus-stamp-stage-05-list.png`
- `kikyo-stamp-stage-05-list.png`

## 確認用一覧

- 確認用一覧には、名前を入れます。
- 一覧は人が見て分かるための画像なので、花名・妖精名を表示してよいです。
- 一覧はアプリ本体では使いません。
- 一覧には日付を入れて、いつ確認したものか分かるようにします。

例:

- `flower-stamp-approved-preview-named-2026-06-18.png`
- `current-fairies-preview-named-2026-06-18.png`
- `hagi-cat-fairy-background-check-2026-06-18.png`

## 保管するセット

完成品を保管する時は、できるだけ次のセットで残します。

- スタンプ本体: 名前なし画像、透明背景またはアプリ用背景
- 元画像: 加工前の画像
- 確認用一覧: 名前入り、見比べ用
- 必要なら背景確認画像: 透明化で欠けがないか確認するための画像

## 花スタンプのルール

- 花スタンプ本体は、花だけが分かる画像にします。
- 画像の中に花名は入れません。
- 確認用一覧には花名を入れます。
- 同じ花が重複して見えないよう、種類ごとに特徴が分かるものを選びます。
- サイズ感は既存の成功品に合わせます。

## 妖精スタンプのルール

- 妖精スタンプ本体は、妖精だけが分かる画像にします。
- 画像の中に妖精名や花名は入れません。
- 確認用一覧には妖精名や花名を入れます。
- 小さい画面でも顔・羽・花の特徴が見えるようにします。
- 透明化する時は、羽・髪・ひげ・細い枝・小花が欠けていないか確認します。

## 勲章のルール

- 現在の完成版は、参加10回と先生の輪（一巡・二巡・三巡・五巡）の5種類です。
- 完成版一覧では「完成版・アプリ使用中」と明記します。
- 将来の勲章は、達成条件・勲章名・表示場所が決まった時点で作ります。
- 用途が決まっていない予備勲章は作りません。
- 新しい勲章を追加する時は、小さい画面での判別、冒険者カード、フクロウの書庫を確認してから完成版へ入れます。

## 保管前チェック

完成品保管庫へ入れる前に、次を確認します。

- 画面で見て成功品と言える
- アプリ用の本体画像に文字が入っていない
- 確認用一覧には名前が入っている
- 背景や透明化で大事な部分が欠けていない
- ファイル名だけで中身が分かる
- 作りかけ画像や失敗画像が混ざっていない
- 完成品一覧表へ、名前・画像・区分を追加している

## 完成品一覧表への追加

- 完成品保管庫へ保存した素材は、同じ作業の中で完成品一覧表にも追加します。
- アプリで使用中の素材と、予備・追加完成品は区分を分けて載せます。
- 一覧には素材名を表示し、どの完成品画像を参照しているか分かるようにします。
- 一覧画像とHTML版がある場合は両方を更新します。
- 一覧の件数表示も、追加後の数に合わせて更新します。
- 更新後は画像の欠け、名前の誤り、古い素材への参照がないか確認します。

## 今日決めた大事なルール

花スタンプ本体・妖精スタンプ本体は、名前なしPNGで保存します。
完成品保管庫の確認一覧は、名前入りで保存します。
