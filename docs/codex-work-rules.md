# Codex 作業ルール

## 推論設定の提案

普段の作業は「中」で進める。

Codex は、次のような場面ではユーザーに「高」への切り替えを提案する。

- 大きな設計方針を決めるとき
- 複雑な不具合の原因を探すとき
- データ構造、保存形式、達成条件など、あとから変えにくい部分を決めるとき
- 複数の画面や機能にまたがる変更をするとき
- 組み合わせ神社のように、汎用性と拡張性が大事な機能を本格的に設計するとき

文言調整、軽い見た目の調整、スクリーンショット確認、小さな相談は基本的に「中」のままでよい。

提案するときは、たとえば次のように短く伝える。

「ここは少し大事な設計なので、高にして考えてもよさそうです。」

「この変更はあとから効いてくるので、高をおすすめします。」

Codex が勝手に切り替えるのではなく、必要そうな場面で提案し、最終的にユーザーが選ぶ。
