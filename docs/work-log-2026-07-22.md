## 最小通信試験の設計

試験対象は、通信・返却・重複防止だけに限定した。

- GitHub Pagesの独立した試験ページからダミーJSONをPOSTする
- `application/json`と`text/plain;charset=UTF-8`を比較する
- どちらも同じダミーJSON文字列を送る
- Apps Script側は`e.postData.contents`をJSON解析する
- 同じ`recordId`はScript Lock内で一件だけ保存する
- 応答を受け取れず再送しても`accepted`を返す
- `storedCount`は試験用`testRunId`名前空間内だけを数える
- `no-cors`は使用しない
- 判定は「合格・条件付き合格・不合格」の3段階とする

Apps Script URLはコードへ直接書かず、試験画面でその都度入力する。URLや送信内容はクエリ文字列へ入れず、URLは端末保存領域にも残さない。

## Apps Script案の安全条件

- Script Propertiesの試験用名前空間だけへ保存する
- Script Lock内で重複確認と保存を行う
- 結果名は`accepted`とする
- 応答遅延は固定ダミー記録だけの短時間試験機能とする
- 削除前に対象キーを一覧表示する
- 一覧確認後、10分間有効な確認トークンを使って削除する
- 一覧確認後に対象キーが変化した場合は削除を中止する
- エラー時に例外文や受信本文を画面へ返さない
