# 管理画面整理 作業記録 2026-07-04

- 管理画面の部品を、当日よく使うもの、慎重に使うもの、ときどき使うもの、非常時だけ使うものに分けた。
- 当日の主役は「参加スタンプ」と「本人スマホに入れるQR」。
- 「対局記録と先生スタンプを反映」は、本人スマホではなく今開いている端末に入る操作なので、QR作成とは分けて見せる必要がある。
- バックアップは現在、スタンプ、獲得済み報酬、操作履歴、先生紹介が中心。対局記録そのものや今日のQR読み取り表示は含まれていないため、画面上で保管範囲を明確にする。
- 詳細は `docs/admin-screen-priority-map-2026-07-04.md` に記録した。
- 参加スタンプも、当日1回だけの受付QRにするとよい。管理画面整理後の次候補として扱う。
- 管理画面に「受付用 参加QRを作る」を追加した。本人スマホで読み取ると、当日の参加スタンプが1つ入り、今日の記録へ着地する。
- 先生ごとの固定QR案も記録した。受付用参加QRと先生席の先生別QRを組み合わせると、当日運営が軽くなる。
- 管理画面に「先生ごとの固定QRを作る」を追加した。今日の日付と先生IDで固定QRを作り、本人スマホで読んだ先生のスタンプが1つ入る。同じスマホで同じ先生QRを読み直しても二重には増えない。
