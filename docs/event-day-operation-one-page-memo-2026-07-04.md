# 8/30イベント 当日運営1枚メモ

状態: 正式保存

初めて触る人に、最初の1回で「今日の記録が花として残った」と見せるための手順。

## 1. 開始前

PC側:

1. 確認用サーバーを起動する。
2. PCで `http://192.168.128.168:4184/` が開けることを確認する。
3. iPhoneでも `http://192.168.128.168:4184/` を開く。
4. 管理画面を開き、パスコードで解除する。
5. 必要なら「記録をリセット」で初期状態にする。

注意:

- URLは必ず `:4184/` まで入れる。
- PCとiPhoneは同じWi-Fiに接続する。
- iPhoneで開けない時は `docs/mobile-preview-connection-runbook-2026-07-04.md` を見る。

## 2. 受付で参加スタンプを入れる

基本は受付用の参加QRを使う。

1. 管理画面の「受付用 参加QRを作る」を押す。
2. 表示されたQRを、参加者本人のスマホで読む。
3. 冒険者カードの「今日の記録」が見えることを確認する。
4. 参加スタンプの花が入っていることを見る。
5. 参加者に「今日の花が入りました。」と伝える。

確認すること:

- 参加スタンプは同じ日に1回だけ。
- 同じQRをもう一度読んでも二重に増えない。
- 管理画面では、反映済みなら「本日は反映済み」になり、ボタンは押せない。
- フォーム回答を見てこの端末へ入れる操作は、QRが使えない時の補助にする。

## 3. 先生スタンプを入れる

基本は先生ごとの固定QRを使う。

1. 管理画面の「先生ごとの固定QRを作る」を押す。
2. 先生名と日付が読めることを確認する。
3. 先生席または受付に、先生ごとのQRを置く。
4. 参加者本人のスマホで、対局した先生のQRを読む。
5. 冒険者カードの「今日の記録」に着地したことを見る。
6. 指導碁スタンプの花が入っていることを確認する。

確認すること:

- 同じ先生QRをもう一度読んでも二重反映されない。
- 先生スタンプは1日3枠まで、今日の記録に花として見える。
- 3枠が埋まった後も、読み取り後は今日の記録が見える位置へ戻る。
- 先生名、日付、QRの置き場所を紙面で間違えない。

## 4. 対局内容も残したい時

ハンデや勝敗まで残したい時だけ、対局内容QRを使う。

1. 管理画面で先生、日付、ハンデ、勝敗を選ぶ。
2. 「本人スマホに入れるQRを作る」を押す。
3. 表示されたQRを、参加者本人のスマホで読む。
4. 冒険者カードの「今日の記録」が見えることを確認する。
5. 参加スタンプ、指導碁スタンプの下に「今日の指導碁」が出ることを確認する。

注意:

- これは本人スマホに入れるためのQR。
- 先生ごとの固定QRでは「今日の指導碁」カードは出ない。固定QRは先生スタンプ用。
- 運営PCで「この端末で確認反映」した分は、本人スマホには入らない。
- QRで本人スマホに入った記録は、運営PCからは戻せない。間違えた時は、そのスマホ側で管理画面を開いて調整する。

## 5. 参加者に見せる

冒険者カードへ移動したら、画面は「今日の記録」へ着地する。

見せる言葉:

> 今日の花が入りました。

対局内容QRを読んだ時:

> 今日の指導碁も記録に残りました。

もう少し説明する時:

> 今日の指導碁が、ここに花で残りました。

見せる場所:

- 参加スタンプ: 花マーク
- 指導碁スタンプ: 先生の花スタンプ台紙
- 対局内容QR: 今日の指導碁カード
- 先生スタンプは1日3枠まで見える

先生の花を押すと、その先生の写真画面へ戻れる。

## 6. 間違えた時

先生を間違えた:

1. まだ本人スマホでQRを読んでいなければ、正しい先生QRを出し直す。
2. 運営PCで確認反映した分なら「直前の先生スタンプを取り消す」を押す。
3. QRで本人スマホに入れた分なら、そのスマホ側の管理画面でスタンプ数を調整する。
4. 正しい先生を選び直す。
5. もう一度、本人スマホで正しいQRを読み取る。

参加スタンプを間違えた:

1. 管理画面の「スタンプ数を調整」で直す。
2. 「確定して保存」を押す。
3. 直近の操作履歴を見る。

不安な時:

- 「直近の操作履歴」を見る。
- 参加者へ見せる前に、冒険者カードの「今日の記録」を確認する。
- 迷ったら新しい操作を増やさず、今日の記録が正しいかを見る。

## 7. 終了後

1. 管理画面で「バックアップを保存」を押す。
2. 保存したファイルを分かる場所へ置く。
3. 紙のQRを回収する。
4. 前日や練習用のQRが残っていないか確認する。
5. 必要なら確認用サーバーを止める。
