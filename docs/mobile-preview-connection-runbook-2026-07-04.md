# スマホ実機確認 接続手順 2026-07-04

状態: 正式保存

8/30イベント前のスマホ実機確認で、iPhoneからアプリを開くための手順。

## 開くURL

iPhoneでは次を開く。

`http://192.168.128.168:4184/`

注意:

- `192.168.128.168` だけでは開かない。
- 必ず `:4184/` まで入れる。
- PCとiPhoneは同じWi-Fiに接続する。

## PC側で確認すること

1. PCのIPアドレスが `192.168.128.168` であることを確認する。
2. 4184番で確認用サーバーが待ち受けていることを確認する。
3. PC内で `http://192.168.128.168:4184/` が開けることを確認する。
4. Windowsファイアウォールで `4184` 番の受信が許可されていることを確認する。

## 今日分かったこと

- Codex内の通常起動では、確認用サーバーがすぐ止まることがあった。
- iPhone確認には、サンドボックス外でサーバーを起動する必要があった。
- Windowsファイアウォールに `Suiyoukai mobile preview 4184` の許可ルールを入れた。
- PC内で開けるのにiPhoneで開けない場合は、ファイアウォールかWi-Fiルーターの端末間通信ブロックを疑う。

## 当日トラブル時の切り分け

### iPhoneで黒い画面のまま

- URLに `:4184/` が入っているか見る。
- `http://` から始まっているか見る。

### 「ページを開けません」になる

- PC側の確認用サーバーが動いているか見る。
- PCで `http://192.168.128.168:4184/` を開けるか見る。
- PCで開けてiPhoneだけ開けない場合は、ファイアウォールかWi-Fi設定を見る。

### PCでは開けるがiPhoneで開けない

- iPhoneが同じWi-Fiにいるか見る。
- ゲストWi-Fiや端末分離のあるWi-Fiではないか見る。
- Windowsファイアウォールの `4184` 許可を確認する。

## 使い終わった後

確認が終わったら、起動した確認用サーバーは止めてよい。

ただし、8/30前のリハーサル中は、スマホで再読み込みできるようにサーバーを起動したままにしておく。
