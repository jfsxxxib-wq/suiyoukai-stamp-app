# 95%の種メモ 2026-07-05

状態: 正式保存

目的: 8/30イベント当日の安心を崩さず、イベント後に「思い出として開ける」アプリへ育てるための種を残す。

## 位置づけ

90%は、当日成功するための品質。

- QRが読める。
- 二重に増えない。
- 今日の記録へ着く。
- 紙のQRが読める。
- スタッフが迷わない。

95%は、イベントが終わったあとも続く品質。

- 今日の記録が、あとで見返せる。
- 花が、ただのスタンプではなく思い出として残る。
- 先生、日付、一局の記憶が自然につながる。
- 参加者が「また来たい」と思える余韻がある。

## 8/30までのルール

8/30までは、95%のために大きな新機能を増やさない。

理由:

- 90%の土台を崩さないため。
- 当日スタッフの説明を増やさないため。
- 参加者がまず「今日の花が入った」と分かることを優先するため。

8/30までにやってよいこと:

- 言葉をそろえる。
- 判断基準を残す。
- 未来の設計メモを書く。
- 実機確認で良かったことを残す。

8/30までに急がないこと:

- 冒険日誌の大きな実装。
- 新しい保存形式の追加。
- 見やすさ設定の実装。
- QR紙面テンプレートの全面作り直し。
- バックアップ対象の拡張。

## 当日の共通声かけ

基本の声かけ:

> 今日の花が入りました。

少し説明する時:

> 今日の指導碁が、ここに花で残りました。

迷った時にスタッフが見る基準:

> 今日の記録に花が見えていれば、まず大丈夫。

この言葉を使う理由:

- 「登録しました」より温かい。
- 「スタンプを押しました」より水曜会らしい。
- 参加者が、自分の一日が残ったと感じやすい。

## 将来の余韻の一文

イベント後にアプリを開いた時、短い一文があるだけで思い出として開きやすくなる。

候補:

- 8月30日、先生との一局が花になりました。
- 今日の一局が、神社の花として残りました。
- あの日の花を、ここで大切にしまっています。
- 先生の輪をひらいた日が、花で残りました。
- 水曜会の一日が、あなたの冒険者カードに咲きました。

使い方の方針:

- 長く説明しない。
- 参加者の名前や個人情報を無理に入れない。
- 先生名や日付を入れる時は、後から安全に扱える形にする。
- 業務記録ではなく、思い出の入口として置く。

## 今日の記録から冒険日誌へつなぐ案

将来、今日の記録がその日の冒険日誌へつながると、95%に近づく。

最小案:

1. 今日の記録に「この日の花を見る」を置く。
2. その日の参加スタンプと先生スタンプをまとめて表示する。
3. 日付ごとに短い余韻の一文を表示する。

まだ決めないこと:

- 日誌を何日分保存するか。
- 先生名、ハンデ、勝敗をどこまで表示するか。
- バックアップに日誌を含めるか。
- 端末変更時にどう引き継ぐか。

判断基準:

- 参加者があとで見てうれしいか。
- スタッフの当日作業が増えすぎないか。
- 個人情報を持ちすぎないか。
- 90%のQR運用を壊さないか。

## 95%へ向けた布石リスト

### 1. 冒険日誌

今日の記録を、日付ごとの思い出として開けるようにする。

最初の到達点:

- 日付。
- 参加スタンプ。
- 先生スタンプ。
- 短い余韻の一文。

### 2. 見やすさ優先設定

老眼の人や、スマホ操作が苦手な人へさらに寄り添う。

最初の到達点:

- 文字を一段階大きくする。
- ボタンや余白を少し広げる。
- 今日の記録をより見つけやすくする。

### 3. QR紙面テンプレート

受付、先生席、予備用で紙面の役割を分ける。

最初の到達点:

- 受付用参加QR。
- 先生席用固定QR。
- 予備・再発行用QR。

### 4. バックアップ拡張

イベント後に安心して残せる範囲を広げる。

最初の到達点:

- 何がバックアップに入るかを画面と紙で一致させる。
- 対局記録明細やQR読み取り履歴を含めるか検討する。

### 5. 失敗時の案内

QR反映に失敗した時、スタッフと参加者が落ち着いて戻れるようにする。

最初の到達点:

- 「もう一度読んでください」
- 「同じ日の記録はすでに入っています」
- 「スタッフに見せてください」

## 残す設計理念

> 90%は、迷わず今日の花が入ること。
> 95%は、その花があとで思い出として開けること。

この二文は、8/30イベント後も判断基準として残す。

機能を増やすか迷ったら、次の順で見る。

1. 当日スタッフが迷わないか。
2. 参加者が今日の花をすぐ見られるか。
3. あとで開いた時に、思い出としてうれしいか。
4. 水曜会らしい温かさが残るか。

## 次にやる時の入口

8/30の実機運用が終わった後、次の順で検討する。

1. 実機確認メモの「良かったこと」を読む。
2. 「今日一番うれしかった瞬間」を読む。
3. 参加者がどこで笑ったか、どこで迷ったかを見る。
4. 冒険日誌、見やすさ設定、QR紙面、バックアップのどれから育てるか決める。

95%は、完成機能の数ではなく、イベント後にもう一度開きたくなる理由で測る。
