# Safariとホーム画面版 保存領域分離の確認

- 実施日: 2026-07-24
- 判定: **対策実装・GitHub保管・公開・公開後実機確認まで合格**
- 原因: Safariとホーム画面版の保存領域分離
- 正式対策: 候補A「Safariでお使いください」
- 通常アプリ: 実装・公開済み
- 自動試験: 全項目合格
- iPhone Safariローカル実機試験・公開後実機確認: 全項目合格
- 保存データへの影響: なし
- GitHub: PR #14を`main`へマージ済み
- GitHub Pages: 公開成功

## 原因

現在のアプリは、名前、受付番号、花、スタンプ、今日の記録をブラウザーの`localStorage`へ保存している。

iPhoneでは通常Safariとホーム画面版が別々の保存領域を使う。両者の`localStorage`は共有されないため、同じ端末でも名前、受付番号、花、スタンプ、今日の記録が別々に見える。

先生ページ、先生共有の15秒タイムアウト対応、サーバー側の問題ではない。

## 実機で確認したこと

- Safariには保存済みの冒険者名、受付番号、参加・先生スタンプが残っていた
- Safari再読み込み後も名前と今日の記録が残った
- 同じ受付QRと先生QRの再読取で二重に増えなかった
- ホーム画面版にはSafariとは異なる名前と受付番号が表示された
- ホーム画面版にはSafariの今日の記録が表示されなかった

Safari側のデータが消えたのではなく、2つの保存場所が分かれている。

## 正式採用した案内

候補Aを正式採用し、通常アプリへローカル実装した。

- 見出し: 「Safariで」「お使いください」
- 主ボタン: 「このまま使う」
- 副ボタン: 「案内を閉じる」
- iPhoneの通常Safariだけに初回表示
- 閉じた履歴は新しい専用キー1件に保存
- 再読み込み、新しいタブ、`?install=1`でも閉じた履歴を尊重
- iPhone版Chrome等、Android、PC、ホーム画面版、管理画面、受付QR、先生QRでは表示しない

既存の名前・受付番号・花・スタンプ・今日の記録の保存キーには触れていない。`manifest.webmanifest`とホーム画面アイコンも変更していない。

ホーム画面アイコンの削除、名前の再入力、端末データのリセット、不要な追加QR読取は行っていない。
