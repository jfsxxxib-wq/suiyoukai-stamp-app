# 先生ページ タイムアウト時サーバー保存実機試験

- 実施日: 2026-07-23
- 判定: 合格
- 対象: 試験専用Apps Scriptと独立試験ページのみ
- 通常アプリ、本番データ、Googleフォーム、本番用スプレッドシート: 未接続・未変更

## 結果

- Apps Scriptに20秒の応答遅延を設定
- ブラウザーは15秒でタイムアウト表示
- 初回送信後のApps Script実行は1回だけで、自動再送なし
- `request-003` / `trial-record-003`はサーバー側で保存済み
- 同じIDを手動再送してもScript Propertyは1件のまま
- 初回受信時刻も変化せず、重複保存なし

## 重要な学び

ブラウザーのタイムアウトは「未保存」を意味しない。今回も画面はタイムアウトしたが、サーバー側では保存されていた。

画面文言は「未保存」ではなく、`送信結果を確認できませんでした`と区別する案が安全。
