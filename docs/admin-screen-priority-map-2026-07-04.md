# 管理画面 整理優先順位メモ 2026-07-04

## 目的

管理画面は、当日よく使う操作と、非常時だけ使う操作が同じ重さで並んでいるため、運営中に「何と何がつながっているか」が分かりにくくなっている。

まずは保存処理やQR反映の仕組みを変えず、画面の見せ方と並び順だけを整理する。今日実機で通った流れを壊さないことを最優先にする。

## 優先順位

### 1. 当日いちばん使う

#### 参加スタンプ

- つながるもの: 参加フォーム確認、お名前入力、今日の参加スタンプ反映、冒険者カードの今日の記録
- 当日役割: 来場確認を見て、参加スタンプを1つ入れる
- 画面上の置き場所: 管理画面の上の方
- 注意: 参加者本人のスマホへ反映したい場合は、最終的には本人スマホ側で見せる必要がある

#### 指導碁スタンプQR

- つながるもの: 対局記録フォーム確認、先生、対局日、ハンデ、勝敗、反映用QR、本人スマホの冒険者カード
- 当日役割: 運営が確認した対局記録を、本人スマホへ入れる
- 画面上の置き場所: 参加スタンプのすぐ下
- 注意: 「反映用QRを作る」が当日の本命ボタン

### 2. 当日使うが、少し慎重に使う

#### この端末だけの確認反映

- 今のボタン名: 対局記録と先生スタンプを反映
- つながるもの: その端末の対局記録、先生スタンプ、今日の記録、直前取り消し、冒険者カードを見る
- 当日役割: 運営端末で画面確認したい時のテスト用、またはその端末を本人用として使っている時だけ使う
- 整理案: 「この端末で確認反映」のように名前を変え、QRより下へ置く
- 注意: 本人スマホには直接入らないので、現在の名前のままだと混乱しやすい

#### 直前の先生スタンプを取り消す

- つながるもの: この端末だけの確認反映で増えた直前1件、対局記録、今日の記録
- 当日役割: 押し間違いをすぐ戻す
- 画面上の置き場所: 「間違えた時」枠
- 注意: QRで本人スマホへ入れた分は、その本人スマホ側の記録なので、運営PCの直前取り消しでは戻らない

### 3. ときどき使う

#### スタンプ数を調整

- つながるもの: 参加スタンプ数、先生ごとの指導碁スタンプ数、報酬、勲章、称号
- 当日役割: 数字がずれた時の手動修正
- 画面上の置き場所: 折りたたみ、または「修正する」枠
- 注意: 数字は直せるが、個別の対局記録や今日の3枠表示とは別扱いなので、通常操作にはしない

#### 直近の操作履歴

- つながるもの: 参加反映、先生反映、取り消し、管理調整、リセット、復元
- 当日役割: 「今何を押したか」を確認する
- 画面上の置き場所: 「間違えた時」枠の下
- 注意: 最新10件だけ見える

### 4. たまに使う

#### 先生紹介を編集

- つながるもの: 花図鑑の先生詳細文
- 当日役割: 基本的には使わない。事前準備で使う
- 画面上の置き場所: 「設定」枠
- 注意: 当日運営の主導線からは外す

#### バックアップ・復元

- つながるもの: スタンプ、獲得済み報酬、操作履歴、先生紹介
- 当日役割: 開始前、終了後、復元が必要な時に使う
- 画面上の置き場所: 「保管・復元」枠
- 注意: 現状のバックアップには、対局記録そのものや今日のQR読み取り表示は含まれていない。画面文言で保管範囲を明確にしたい

### 5. 非常時だけ使う

#### 記録をリセット

- つながるもの: スタンプ、対局記録、確認待ち記録、QR二重読み取り履歴、今日の3枠表示
- 当日役割: テスト後の初期化、または完全にやり直す時
- 画面上の置き場所: 最下部の危険操作枠
- 注意: 当日中に誤って押さないよう、今の確認入力は維持する

## 現在いちばん混乱しやすい接続

### 「反映」と「QR」の違い

- 「対局記録と先生スタンプを反映」: 今開いている端末に反映する
- 「反映用QRを作る」: 参加者本人のスマホに反映するためのQRを作る

当日運営では、本人スマホに入れることが大事なので「反映用QRを作る」を主役にする。

### Googleフォーム確認との関係

- 参加フォーム: 名前を見て、参加スタンプ反映へ進む
- 対局記録フォーム: 先生、対局日、ハンデ、勝敗を見て、QR作成へ進む

対局記録フォームに名前欄がないため、現状は「誰のスマホでQRを読むか」で本人と結びつける運用になる。将来的には、対局記録フォームにも名前か受付番号を入れると、混雑時の取り違えが減る。

## 画面整理案

### 上から並べる順番

1. 今日の運営
2. 参加スタンプ
3. 指導碁スタンプQR
4. 間違えた時
5. 手動調整
6. 操作履歴
7. 設定
8. 保管・復元
9. 危険操作

### ボタン名の見直し候補

- 「反映用QRを作る」 -> 「本人スマホに入れるQRを作る」
- 「対局記録と先生スタンプを反映」 -> 「この端末で確認反映」
- 「確定して保存」 -> 「手動調整を保存」
- 「記録をリセット」 -> 「すべての記録をリセット」

## 実装するならこの順番

1. 見出しと説明だけを整理する。保存処理は触らない
2. 指導碁ではQRボタンを主役にし、端末確認用ボタンを下へ移す
3. 手動調整、先生紹介、バックアップ、リセットを折りたたみ候補にする
4. バックアップの説明に「対局記録そのものは含まない」を明記する
5. 対局記録フォームに名前または受付番号を入れるか検討する

## 次の追加候補: 参加スタンプQR

参加スタンプもQR方式にする。

- 当日用に1つの受付QRを先に作っておく
- 受付が済んだ人から本人スマホで読み取る
- その日の参加スタンプが1つ入る
- 同じ日には二重に増えない
- 冒険者カードの「今日の記録」へ着地する

参加スタンプは当日1つと決まっているため、先生スタンプQRよりも受付運用に向いている。管理画面整理後の次の実装候補にする。

## 次の追加候補: 先生ごとの固定QR

名前入力を使わないなら、当日参加する先生ごとのQRを先に用意しておく運用もよい。

- 先生ごとに1枚のQRを作る
- 参加者は指導碁を受けた先生のQRを本人スマホで読み取る
- その先生の花スタンプが1つ入る
- 同じQRを同じスマホで読み直しても二重には増えない
- ハンデや勝敗まで残す場合は、今の対局記録フォーム確認QRを使う

当日の軽さを優先するなら、受付用の参加QRと、先生席に置く先生別QRの組み合わせが分かりやすい。対局内容の細かい記録を残したい時だけ、フォーム確認からQRを作る運用にする。

## Codex君のおすすめ

次に触るなら、まずはコードの中身を大きく変えずに、管理画面の文章と並び順だけを直すのがよい。

理由は、今日やっと実機でQR、3枠表示、リセットが通ったため。仕組みを動かす前に、画面上で「当日はこの順に押せばよい」と見えるようにする方が安全。
