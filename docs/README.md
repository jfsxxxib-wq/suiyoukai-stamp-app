# 水曜会 花スタンプ冒険記録アプリ

## このプロジェクトについて

水曜会で使う、スマホ向けの花スタンプ・冒険記録アプリです。

目的は、会への参加や先生との指導碁を、ポイントカードではなく「冒険記録帳」のように楽しく残すことです。御朱印の記念感、花図鑑の収集感、冒険者カードの成長感を組み合わせます。

## 主要ファイル

- `index.html`
  - 画面構成です。
  - トップ、花図鑑、冒険者カード、参加スタンプ、先生画面、運営用入口などがあります。

- `styles.css`
  - 見た目の調整です。
  - スマホ向けの表示、先生カード、達成モーダル、冒険者カードなどのデザインが入っています。

- `script.js`
  - 画面の動きです。
  - タブ切り替え、参加スタンプと先生スタンプの保存、5回達成モーダル、冒険者カード更新などを担当します。

- `stamp-rules.js`
  - ルール定義と達成判定です。
  - 参加スタンプ、先生ごとの妖精達成、先生の輪の判定をまとめています。

- `assets/`
  - 花、妖精、先生写真、勲章、地図などの画像素材です。

- `docs/`
  - このプロジェクトの説明、ルール、作業メモ、次にやることです。

## 現在の状態

- 1ファイルで開けるHTMLアプリとして作っています。
- 先生ごとのスタンプは、`localStorage` に保存できます。
- 参加スタンプの保存入口があります。
- 参加スタンプと指導碁スタンプは、確定直前に運営用パスコードが必要です。
- 運営認証後はその1件だけを保存し、自動で再ロックします。
- 管理画面には直近10件の操作履歴を、操作種別・対象・変更前後・日時付きで表示します。端末内には最大50件保存します。
- 地図には、現在の保存記録から最も近い達成を一件選ぶ「次の冒険」を表示します。
- 「次の冒険」の右側には小さな妖精案内人を表示し、獲得後は最後に出会った妖精が案内役になります。
- 運営画面から、この端末に保存した記録をリセットできます。誤操作防止のため、実行時は「リセット」と入力します。
- 運営画面では、参加スタンプと先生ごとの指導碁スタンプを下書きで調整し、「確定して保存」を押した時だけ保存します。
- スタンプ数を減らしても、一度獲得した妖精、勲章、称号、特別な仲間、花図鑑の発見は冒険の記憶として残ります。
- 運営画面の確定保存と全リセットも、実行後に自動で再ロックします。
- 運営画面から、スタンプ数・獲得済み報酬・操作履歴をJSONファイルへバックアップし、安全確認後に復元できます。
- 運営画面は簡易パスコードでロックしています。現在のパスコードは `suiyoukai2026` です。
- 先生スタンプと参加スタンプは、巡が進むと花と妖精が変わります。
- 先生スタンプは、先生5人 × 3巡 = 15妖精まで判定できます。
- 参加スタンプは、10回ごとに1巡進み、3巡 = 3妖精まで判定できます。
- 花スタンプは18枠すべて別の花になるようにしました。
- 追加した12種類は、藤、金木犀、蓮、菫、牡丹、百合、朝顔、桔梗、撫子、水仙、萩、芍薬です。
- 先生ごとの5回達成で、別モーダルに妖精達成を表示します。
- 冒険者カードには、現在の進み具合と、これまでに獲得した妖精、勲章、称号を表示します。
- ルール定義は `stamp-rules.js` にあります。

## 開き方

`index.html` をブラウザで開きます。

表示が古い場合は、ブラウザで `Ctrl + F5` を押して再読み込みしてください。

## バックアップと復元

1. 管理画面をパスコードで解除します。
2. 「バックアップを保存」で端末へファイルを保存します。
3. 復元時は「バックアップから復元」でファイルを選び、表示された保存日時・参加回数・先生記録・獲得数・履歴件数を確認します。
4. 「復元」と入力して実行します。現在の状態は、復元前バックアップとして先に自動保存されます。

対応していない形式や壊れたファイルは復元できません。バックアップには運営パスコードや参加者の個人情報を含めません。

## 注意

- 先生ごとのスタンプはこの端末のブラウザに保存されます。外部データベース連携はまだ入れていません。
- 画像素材を差し替える時は、`assets/` 内のファイル名と `stamp-rules.js` / `script.js` の参照名をそろえてください。
- 先生ごとの妖精達成と、先生の輪の一巡達成は別ルールです。
